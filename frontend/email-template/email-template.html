Setup Explanation

1. Static Web Page on Azure

The front-end of this application is a static webpage hosted on Azure Storage. The webpage contains a contact form where users can enter their email. This form sends a POST request to a backend API when submitted.

2. Backend - Node.js in Docker

The backend API that handles the form submission is built with Node.js and runs inside a Docker container. The container is responsible for receiving the POST request from the static webpage and sending an email.

3. Email Sending with SendGrid

To send the emails, the application integrates with SendGrid as the email service provider. SendGrid provides an API that the application uses to send emails. The API key is securely stored in the environment variable, which is loaded into the app using a .env file.

4. Docker Container Setup

The backend API is packaged into a Docker image. The container is exposed on port 3000, which is configured to handle the requests from the static webpage.

5. CI/CD Pipeline for Deployment

The application is deployed using a Continuous Integration and Continuous Deployment (CI/CD) pipeline, which automatically updates the application whenever changes are pushed to the GitHub repository.

6. Local Development and Testing

During the development process, the static webpage and backend API are tested locally. The Docker container is run on a virtual machine, and the static webpage is hosted using Visual Studio Code's Live Server.
